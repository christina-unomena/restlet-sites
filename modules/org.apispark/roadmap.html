Want to know the features that are coming up next and the latest enhancements released on the platform? This page will contain up-to-date information. In addition, make sure to check out our blog.

Your feed-back is always welcome. Please feel free to contact us.

February 2014 release (in development)

  • New user experience (UI, guide)
  • Cross-region fail-over (with geo-DNS routing)
  • Global CDN for better latency
  • Integration with Google Cloud, Runscope and API Commons
  • API firewall (white & black lists, rate limitation)

December 2013 release (in production)

  • HTTPS endpoints for APISpark.net subdomains
  • Quick export of Data Stores as Web APIs
  • Revamped global reordering in console
  • Fixed 18 issues
  • Public beta announced at APIDays Paris

October 2013 release

  • Better documentation (user guide, samples in dashboard)
  • HTTPS protection for APISpark console and deployed APIs
  • Updated iOS client SDK (complex representations)
  • Extended private beta to all sign ups

September 2013 release

  • More downloads (GWT and Android client SDKs, code export)
  • Better documentation (user guide, more tutorials)
  • Reworked login dialog, adding support for GitHub
  • Subscription management (plan selection, payment, billing)
  • Deployment region selection (US-West or EU-West)

Summer 2013 release

  • External Data Store wrappers (Google Spreadsheet, Parse, StackMob, Firebase, AWS S3, JDBC)
  • Complex entities and representations (beta)
  • Stabilized overall platform, fixing about 50 issues
  • Pricing of premium plans and extra credits announced
  • Source code export (based on Restlet Framework)

April 2013 release

  • Intra-region fail-over (APIs and entity stores)
  • Infrastructure upgrade (migration to Cassandra 1.2)
  • Complex entities and representations (unstable)
  • Enhanced mapping (source selection, Java code generation)
  • Composite APIs (API import, optimized local calls)

March 2013 release

  • Full cell deletion (freeing the subdomain name)
  • Revert 'Published' APIs to 'Draft' possible (if has no external user)
  • Custom domain names (additional endpoints)
  • Faster API deployment times (separate downloads generation)
  • Improved overall stability (UI, deployment and runtime)

February 2013

  • Local entity stores, based on Apache Cassandra
  • Local file stores, based on AWS S3
  • API creation, hosting, management and usage
  • Basic API promotion in the Catalog area
